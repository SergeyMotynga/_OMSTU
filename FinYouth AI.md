**F**inancial **I**ntelligence for **Y**outh  
*Искусственный интеллект для финансовой грамотности молодёжи*  
**Telegram-бот | MVP за 3 дня | Чемпионат**

---

## 1. Описание продукта

**FinYouth AI** — интерактивный ИИ-ассистент в Telegram, который помогает молодёжи 16–25 лет понять финансы через геймификацию, персонализацию и анализ трат.

| Параметр            | Описание |
|---------------------|---------|
| **Название**        | FinYouth AI |
| **Расшифровка**     | **F**inancial **I**ntelligence for **Y**outh |
| **Формат**          | Telegram-бот |
| **Целевая аудитория** | Молодёжь 16–25 лет |
| **Проблема**        | Низкая финансовая грамотность: импульсивные траты, долги, страх инвестиций |
| **Решение**         | ИИ-коуч с мемами, викторинами, симуляциями и анализом чеков |
| **Уникальность**    | Российский ИИ (GigaChat), OCR чеков, ежедневные советы без повторов |

## 2. Функционал (MVP — 10 функций)

| № | Функция                  | Описание | Пример |
|---|--------------------------|---------|--------|
| 1 | **/start**               | Мотивирующее приветствие + кнопка «Начать» | _"Многие не знают, куда деваются деньги — но ты можешь это изменить!"_ |
| 2 | **Персональный опрос**   | 4–5 шагов → конкретная цель + учёт долгов | _"На что копишь? → iPhone 16 за 120 000 ₽"_ |
| 3 | **Главное меню**         | 6 кнопок: План, Викторина, Симулятор, Советы, Прогресс, Помощь | — |
| 4 | **Викторины**            | 5 тем, баллы, значки | _"Что такое сложный процент?" → +10 баллов_ |
| 5 | **Симулятор инвестиций** | Гибкие %, 3 сценария (пессимистичный, средний, оптимистичный) | _"10 000 ₽ в акции → от +5% до +20%"_ |
| 6 | **Анализ чеков**         | Фото/текст → OCR → **GigaChat определяет категорию по списку (без ключевых слов)** → кэш + обратная связь → псевдо-самообучение | _"Кофе 350 ₽ → Еда вне дома (кэшировано)"_ |
| 7 | **Ежедневные советы**    | Без повторов, из базы 30+ | _"Откладывай 10% с подработки"_ |
| 8 | **Прогресс**             | Баллы, значки, статистика трат | _"Финансовый ниндзя: 140 баллов"_ |
| 9 | **Свободный чат**        | GigaChat отвечает на любые вопросы | _"Что лучше — акции или депозит?"_ |
| 10| **Отказ от ответственности** | В `/help` и при старте | _"Не финансовая консультация"_ |

## 3. Технологический стек

| Компонент         | Технология                  | Обоснование |
|-------------------|-----------------------------|-----------|
| **Язык**          | Python 3.10+                | Универсальность, ИИ, боты |
| **Бот**           | **Aiogram 3.x**             | Асинхронность, FSM, кнопки |
| **ИИ (основной)** | **GigaChat API (Сбер)**     | Российский, бесплатно до 1M токенов, отличный русский |
| **ИИ (резерв)**   | **YandexGPT**               | Бесплатно до 100K токенов/день |
| **БД**            | **SQLite**                  | Локальная, без сервера |
| **OCR**           | **pytesseract + Pillow**    | Распознавание чеков (русский) |
| **Графики**       | **Matplotlib**              | PNG → Telegram |
| **Планировщик**   | **APScheduler**             | Ежедневные советы |
| **Деплой**        | **Heroku**                  | Бесплатно, 2 клика |
| **Разработка**    | VS Code, GitHub, Pytest     | Контроль версий, тесты |
| **Самообучение**  | **Кэш + обратная связь**    | Кэширование ответов + кнопки "Правильно/Нет" → рост точности |

## 4. Архитектура

Пользователь → Telegram

↓

Aiogram Бот → Webhook (Heroku)

↓

Обработчики

├── Опрос → SQLite

├── Викторина → JSON + SQLite

├── Симулятор → Matplotlib

├── OCR → pytesseract

├── Советы → APScheduler

└── Чат → GigaChat API (резерв: YandexGPT)

↓

Ответ → Telegram

## 5. База данных (SQLite)

```sql
-- Пользователи
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    telegram_id INTEGER UNIQUE,
    age INTEGER,
    income INTEGER,
    expenses INTEGER,
    goal_name TEXT,
    goal_amount INTEGER,
    debt INTEGER,
    points INTEGER DEFAULT 0,
    badge TEXT DEFAULT 'Новичок'
);

-- Расходы
CREATE TABLE expenses (
    id INTEGER PRIMARY KEY,
    user_id INTEGER,
    amount INTEGER,
    category TEXT,
    date TEXT
);

-- Показанные советы
CREATE TABLE shown_tips (
    user_id INTEGER,
    tip_id INTEGER
);

-- Кэш категорий (для псевдо-самообучения)
CREATE TABLE category_cache (
    text_hash TEXT PRIMARY KEY,
    original_text TEXT,
    category TEXT,
    confidence REAL,
    source TEXT  -- 'gigachat' или 'user_feedback'
);

-- Обратная связь
CREATE TABLE user_feedback (
    cache_id TEXT,
    correct_category TEXT,
    timestamp TEXT
);
```
