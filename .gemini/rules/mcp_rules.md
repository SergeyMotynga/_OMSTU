# Правила использования MCP Протоколов

Этот документ — руководство по использованию доступных MCP серверов и их протоколов. Я буду использовать его как основу для принятия решений при выполнении твоих задач.

## 1. `filesystem` — Работа с файлами

Этот сервер предоставляет доступ к локальной файловой системе. Я буду использовать его для любых операций с файлами и папками.

- **`filesystem.readFile`**: Для чтения содержимого файла.
- **`filesystem.writeFile`**: Для создания или перезаписи файла.
- **`filesystem.directoryTree`**: Для получения дерева файлов и папок.
- **`filesystem.listDirectory`**: Для просмотра содержимого одной папки.
- **`filesystem.createDirectory`**: Для создания новой папки.
- **`filesystem.deleteFile` / `filesystem.deleteDirectory`**: Для удаления файлов или папок.
- **`filesystem.moveFile`**: Для перемещения или переименования.

**Когда использовать**: При любых запросах, связанных с чтением, созданием, изменением или удалением файлов и папок в проекте.

## 2. `github` — Интеграция с GitHub

Полный набор инструментов для взаимодействия с GitHub.

- **Поиск**: `github.search_code`, `github.search_repositories`.
- **Работа с репозиторием**: `github.get_file_contents`, `github.create_or_update_file`, `github.create_branch`.
- **Работа с Issues**: `github.create_issue`, `github.get_issue`, `github.list_issues`.
- **Работа с Pull Requests**: `github.create_pull_request`, `github.get_pull_request`, `github.merge_pull_request`.

**Когда использовать**: Когда задача требует взаимодействия с GitHub: поиск кода, чтение файлов из репозитория, создание PR, управление задачами (issues).

## 3. `playwright` / `webresearch` — Работа с вебом

Два сервера для взаимодействия с интернетом.

- **`playwright` (сервер `browser`)**: Для продвинутой автоматизации браузера.
  - **`browser.navigate`**: Переход на URL.
  - **`browser.click` / `browser.fill`**: Взаимодействие с элементами страницы.
  - **`browser.screenshot`**: Создание скриншотов.
- **`webresearch` (сервер `google`)**: Для поиска информации.
  - **`google.search`**: Поиск в Google.
  - **`google.visit_page`**: Получение содержимого страницы.

**Когда использовать**: `playwright` — для задач, требующих прямого взаимодействия с сайтами (залогиниться, нажать кнопки, заполнить формы). `webresearch` — для поиска информации и анализа содержимого веб-страниц.

## 4. `npm-helper` / `npm-sentinel` — Работа с NPM

Два сервера для анализа и управления зависимостями Node.js.

- **`npm-helper`**:
  - **`npm.search`**: Поиск пакетов.
  - **`npm.get_package_details`**: Получение детальной информации о пакете.
- **`npm-sentinel`**:
  - **`check_updates`**: Проверка на наличие устаревших пакетов.
  - **`upgrade_packages`**: Автоматическое обновление пакетов.

**Когда использовать**: При анализе `package.json`, поиске информации о пакетах, проверке и обновлении зависимостей.

## 5. `context7` — Работа с документацией

- **`context7.resolve_library_id`**: Найти правильный ID библиотеки.
- **`context7.get_library_docs`**: Получить документацию по ID библиотеки.

**Когда использовать**: Когда для решения задачи мне нужна информация из документации по какой-либо библиотеке или фреймворку.

## 6. `sequential-thinking` — Комплексное планирование

- **`sequentialthinking.sequentialthinking`**: Разбить сложную задачу на последовательность шагов, проанализировать их и выстроить план.

**Когда использовать**: Для нетривиальных задач, требующих декомпозиции и пошагового выполнения с возможностью коррекции плана. Например: "спроектируй архитектуру простого веб-приложения" или "проанализируй и исправь сложный баг".

## 7. `everything` — Общие утилиты

- **`run_shell_command`**: Выполнение любой команды в терминале. Это один из самых важных протоколов.
- **`save_memory`**: Запомнить информацию для будущих сессий.

**Когда использовать**: `run_shell_command` — для запуска тестов, сборки проекта, работы с git, установки зависимостей и т.д. `save_memory` — когда ты прямо просишь меня что-то запомнить.
