## Теоретическая часть

### Что такое кластеризация текстов?

**Кластеризация** — это метод машинного обучения без учителя (unsupervised learning), который группирует похожие объекты в кластеры.

**Применение к текстам:**
- Группировка новостей по темам
- Сегментация клиентских отзывов
- Организация документов
- Обнаружение трендов в социальных сетях

### Векторизация текстов

Компьютеры не понимают текст напрямую. Нужно преобразовать слова в числа (векторы).

#### TF-IDF (Term Frequency - Inverse Document Frequency)

**Идея:** Важность слова = частота в документе × редкость во всех документах

**Формула:**
```
TF-IDF(word, doc) = TF(word, doc) × IDF(word)

TF(word, doc) = количество раз слово встречается в документе
IDF(word) = log(общее количество документов / количество документов со словом)
```

**Пример:**

Документы:
- Doc 1: "кот ест рыбу"
- Doc 2: "собака ест мясо"
- Doc 3: "кот спит"

Для слова "кот" в Doc 1:
```
TF("кот", Doc1) = 1/3 = 0.33
IDF("кот") = log(3/2) = 0.18
TF-IDF = 0.33 × 0.18 = 0.06
```

Для слова "ест" в Doc 1:
```
TF("ест", Doc1) = 1/3 = 0.33
IDF("ест") = log(3/2) = 0.18  (встречается в 2 документах)
TF-IDF = 0.33 × 0.18 = 0.06
```

**Особенности:**
- ✅ Быстро работает
- ✅ Не требует обучения
- ✅ Интерпретируемо
- ❌ Не учитывает контекст ("банк" как финансовое учреждение vs берег реки)
- ❌ Разреженные векторы (много нулей)

#### SBERT (Sentence-BERT)

**Идея:** Нейросеть (BERT) преобразует текст в плотный вектор, учитывая контекст.

**Как работает:**
1. Текст разбивается на токены
2. Каждый токен преобразуется в вектор
3. BERT обрабатывает всю последовательность
4. Получаем вектор фиксированной длины (384 или 768)

**Преимущества:**
- ✅ Учитывает семантику (смысл)
- ✅ Похожие по смыслу фразы → похожие векторы
- ✅ Работает с синонимами
- ❌ Медленнее TF-IDF
- ❌ Требует больше памяти

**Пример:**
```
"Кот ловит мышь" → [0.2, -0.5, 0.8, ..., 0.3]  (384 числа)
"Кошка охотится на грызуна" → [0.19, -0.48, 0.82, ..., 0.29]  (похожий вектор!)
```

### Сравнение TF-IDF vs SBERT

| Характеристика | TF-IDF | SBERT |
|---------------|--------|-------|
| Размерность | 5000-10000 | 384-768 |
| Разреженность | Много нулей | Плотный вектор |
| Семантика | Нет | Да |
| Скорость | Быстро | Медленно |
| Память | Мало | Много |
| Синонимы | Не понимает | Понимает |

#### 1. Удаление HTML-тегов

```python
from bs4 import BeautifulSoup
text = BeautifulSoup(text, 'html.parser').get_text()
```

**Зачем?**
```
До:  "This is <b>great</b> product!"

```

#### 2. Удаление эмодзи

```python
import re
emoji_pattern = re.compile("[...]", flags=re.UNICODE)
text = emoji_pattern.sub(r'', text)
```

**Зачем?**
```
До:  "Love it! 😍❤️"
После: "Love it!"
```

#### 3. Приведение к нижнему регистру

```python
text = text.lower()
```

**Зачем?**
```
До:  "Coffee", "coffee", "COFFEE" — 3 разных слова
После: "coffee", "coffee", "coffee" — 1 слово
```

#### 4. Удаление пунктуации и цифр

```python
text = re.sub(r'[^a-z\s]', ' ', text)
```

**Зачем?**
```
До:  "Great! Product... 5/5"
После: "great product"
```

#### 5. Токенизация

```python
from nltk.tokenize import word_tokenize
tokens = word_tokenize(text)
```

**Результат:**
```
"hello world" → ["hello", "world"]
```

#### 6. Удаление стоп-слов

```python
from nltk.corpus import stopwords
stop_words = set(stopwords.words('english'))
tokens = [word for word in tokens if word not in stop_words]
```

**Стоп-слова:** the, a, an, is, are, was, were, to, of, in, ...

**Зачем?**
```
До:  "the coffee is very good"
После: "coffee very good"
```

Стоп-слова не несут смысловой нагрузки, но занимают место в векторе.

#### 7. Лемматизация

```python
from nltk.stem import WordNetLemmatizer
lemmatizer = WordNetLemmatizer()
tokens = [lemmatizer.lemmatize(word) for word in tokens]
```

**Что делает?**
```
"running" → "run"
"better" → "good"
"cats" → "cat"
```

Приводит слова к базовой форме.

**Итоговый пример:**

```
Исходный текст:
"Having tried a couple of other brands of gluten-free sandwich cookies,
these are the best of the bunch! They're crunchy and true to the texture."

После предобработки:
"tried couple brand gluten free sandwich cooky best bunch crunchy true texture"
```

## Алгоритмы кластеризации

### K-Means

**Идея:** Разделить данные на k групп так, чтобы точки внутри группы были близко друг к другу.

**Алгоритм:**

1. **Инициализация:** Случайно выбрать k центроидов (центров кластеров)

2. **Назначение:** Каждую точку отнести к ближайшему центроиду
   ```
   Точка A: distance(A, centroid_1) = 3.2
   Точка A: distance(A, centroid_2) = 1.5  ← ближайший!
   Точка A: distance(A, centroid_3) = 4.1

   A → Кластер 2
   ```

3. **Обновление:** Пересчитать центроиды как среднее точек в кластере
   ```
   Кластер 1: [точка1, точка2, точка3]
   Новый центроид = (точка1 + точка2 + точка3) / 3
   ```

4. **Повторять** шаги 2-3 до сходимости

**Метрика расстояния:**
```python
distance = sqrt((x1-x2)² + (y1-y2)² + ... + (zn-zn)²)
```

Евклидово расстояние в многомерном пространстве.

**Как выбрать k (количество кластеров)?**

#### Метод локтя (Elbow Method)

```
Inertia (сумма квадратов расстояний до центроидов):

k=2:  10000
k=3:  7000
k=4:  5000
k=5:  4500  ← "локоть"
k=6:  4300
k=7:  4200
```

График имеет форму "локтя" — после k=5 улучшение замедляется.

#### Silhouette Score

```
Silhouette Score = (b - a) / max(a, b)

a = среднее расстояние до точек своего кластера
b = среднее расстояние до точек ближайшего другого кластера
```

**Интерпретация:**
- **+1:** Идеально (точки очень близко к своему кластеру, далеко от других)
- **0:** Точки на границе кластеров
- **-1:** Точки попали не в тот кластер

**Выбираем k с максимальным Silhouette Score.**

### Ячейка 9: Функция предобработки

```python
def preprocess_text(text):
    # 1. Удаление HTML
    text = BeautifulSoup(text, 'html.parser').get_text()

    # 2. Удаление эмодзи
    emoji_pattern = re.compile("[...]")
    text = emoji_pattern.sub(r'', text)

    # 3. Lowercase
    text = text.lower()

    # 4. Только буквы
    text = re.sub(r'[^a-z\s]', ' ', text)

    # 5. Убрать лишние пробелы
    text = re.sub(r'\s+', ' ', text).strip()

    # 6. Токенизация
    tokens = word_tokenize(text)

    # 7. Стоп-слова + короткие слова
    tokens = [word for word in tokens if word not in stop_words and len(word) > 2]

    # 8. Лемматизация
    tokens = [lemmatizer.lemmatize(word) for word in tokens]

    return ' '.join(tokens)
```

### Ячейка 19: Определение оптимального k

```python
def find_optimal_clusters(data, max_k=10):
    inertias = []
    silhouette_scores = []

    for k in range(2, max_k + 1):
        kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
        kmeans.fit(data)

        inertias.append(kmeans.inertia_)
        silhouette_scores.append(silhouette_score(data, kmeans.labels_))
```

**Что делаем:**
1. Перебираем k от 2 до 10
2. Для каждого k запускаем K-Means
3. Считаем inertia и silhouette score
4. Строим графики

**Inertia (инерция):**
```python
inertia = sum((point - centroid)²)
```

Сумма квадратов расстояний от точек до их центроидов.
- Меньше → лучше
- Но всегда уменьшается с ростом k!

**Silhouette Score:**
```python
score = mean((b - a) / max(a, b))  # для всех точек
```

- Больше → лучше
- Имеет максимум при оптимальном k

**Визуализация:**

График 1: Метод локтя (Inertia vs k)
```
Inertia
  │
  │●
  │ ●
  │  ●
  │   ●___●___●___  ← после локтя почти не падает
  └──────────────── k
     2  3  4  5  6
```

График 2: Silhouette Score vs k
```
Score
  │      ●
  │     ╱ ╲
  │    ╱   ╲
  │   ●     ●     ← максимум = оптимальный k
  └──────────────── k
     2  3  4  5  6
```

**Выбор k:**

```python
optimal_k = k_range[np.argmax(silhouette_scores)]
```

Берём k с максимальным Silhouette Score.

### Ячейка 31-34: Метрики качества кластеризации

#### Silhouette Score

```python
silhouette = silhouette_score(embeddings, labels)
```

**Формула:**
```
s(i) = (b(i) - a(i)) / max(a(i), b(i))

a(i) = среднее расстояние от точки i до других точек своего кластера
b(i) = среднее расстояние от точки i до точек ближайшего другого кластера
```

**Диапазон:** [-1, 1]
- **1.0:** Идеально (точки очень близко к своему кластеру)
- **0.5-0.7:** Хорошо
- **0.0:** Точки на границе кластеров
- **<0:** Точки скорее всего в неправильном кластере

#### Davies-Bouldin Index

```python
davies_bouldin = davies_bouldin_score(embeddings, labels)
```

**Формула:**
```
DB = (1/k) × sum( max( (s_i + s_j) / d(c_i, c_j) ) )

s_i = среднее расстояние от точек кластера i до центроида i
d(c_i, c_j) = расстояние между центроидами i и j
```

**Диапазон:** [0, ∞)
- **Меньше → лучше**
- **<1:** Хорошо (кластеры компактные и далеко друг от друга)
- **>2:** Плохо (кластеры перекрываются)

#### Calinski-Harabasz Index

```python
calinski = calinski_harabasz_score(embeddings, labels)
```

**Формула:**
```
CH = (between-cluster variance / within-cluster variance) × (n - k) / (k - 1)
```

**Диапазон:** [0, ∞)
- **Больше → лучше**
- Измеряет, насколько плотные кластеры и насколько они разделены

**Интерпретация метрик:**

```python
# Пример хорошей кластеризации:
Silhouette:          0.7123    ✅ Высокий
Davies-Bouldin:      0.8456    ✅ Низкий
Calinski-Harabasz:   3456.78   ✅ Высокий

# Пример плохой кластеризации:
Silhouette:          0.1234    ❌ Низкий
Davies-Bouldin:      2.3456    ❌ Высокий
Calinski-Harabasz:   234.56    ❌ Низкий
```

### Ячейка 36-37: Извлечение топ-слов (TF-IDF)

```python
def get_top_words_tfidf(tfidf_matrix, labels, feature_names, n_clusters, top_n=15):
    for cluster_id in range(n_clusters):
        # Находим документы кластера
        cluster_indices = np.where(labels == cluster_id)[0]

        # Усредняем TF-IDF векторы
        cluster_center = tfidf_matrix[cluster_indices].mean(axis=0).A1

        # Топ-15 слов по весу
        top_indices = cluster_center.argsort()[-top_n:][::-1]
        top_words = [(feature_names[i], cluster_center[i]) for i in top_indices]
```

**Что происходит:**

1. **Находим документы кластера:**
   ```python
   cluster_indices = np.where(labels == 0)[0]  # [5, 12, 34, 67, ...]
   ```

2. **Усредняем векторы:**
   ```python
   # TF-IDF матрица для кластера:
   [[0.0, 0.3, 0.0, 0.8, ...],   # документ 5
    [0.1, 0.2, 0.0, 0.7, ...],   # документ 12
    [0.0, 0.4, 0.1, 0.9, ...]]   # документ 34

   # Среднее:
   [0.033, 0.3, 0.033, 0.8, ...]  # для каждого слова
   ```

3. **Сортируем по весу:**
   ```python
   top_indices = [3, 1, 0, 2, ...]  # индексы в порядке убывания веса
   ```

4. **Получаем слова:**
   ```python
   top_words = [
       ('taste', 0.8),
       ('flavor', 0.3),
       ('good', 0.15),
       ...
   ]
   ```

**Почему это работает:**
- Если слово важно для кластера, оно часто встречается в его документах
- Усреднение TF-IDF весов показывает "типичность" слова для кластера

### Ячейка 40-41: Извлечение топ-слов (KeyBERT)

```python
def get_top_words_keybert(texts, labels, n_clusters, kw_model, top_n=15):
    for cluster_id in range(n_clusters):
        cluster_indices = np.where(labels == cluster_id)[0]

        # Объединяем все тексты кластера
        cluster_text = ' '.join([texts[i] for i in cluster_indices])

        # Извлекаем ключевые слова
        keywords = kw_model.extract_keywords(
            cluster_text,
            keyphrase_ngram_range=(1, 2),
            top_n=top_n,
            use_maxsum=True,
            diversity=0.5
        )
```

**Что такое KeyBERT?**

Библиотека для извлечения ключевых слов с помощью BERT.

**Как работает:**
1. Текст кодируется в вектор (SBERT)
2. Каждое слово/фраза кодируется в вектор
3. Считается cosine similarity между текстом и словами
4. Слова с наибольшей похожестью = ключевые слова

**Параметры:**

**`keyphrase_ngram_range=(1, 2)`**
- Униграммы и биграммы
- Может найти "green tea", а не только "green" и "tea"

**`use_maxsum=True`**
- Использует алгоритм Max Sum Distance
- Находит разнообразные ключевые слова (не все синонимы)

**`diversity=0.5`**
- Баланс между релевантностью и разнообразием
- 0 = только самые релевантные (могут повторяться)
- 1 = максимальное разнообразие (могут быть менее релевантными)

**Результат:**

```python
keywords = [
    ('coffee', 0.72),
    ('coffee cup', 0.68),
    ('brew', 0.65),
    ('roast', 0.61),
    ...
]
```

**Отличие от TF-IDF:**
- TF-IDF: статистический подход (частота слов)
- KeyBERT: семантический подход (смысл слов)

## Вопросы для защиты

### Базовые вопросы

**1. Что такое TF-IDF и как он работает?**

**Ответ:**
TF-IDF (Term Frequency - Inverse Document Frequency) — это метод векторизации текста. Он оценивает важность слова через две компоненты:
- **TF:** частота слова в документе
- **IDF:** редкость слова во всех документах

Формула: `TF-IDF = TF × log(N / df)`, где N — общее количество документов, df — количество документов со словом.

Слова, которые часто встречаются в документе, но редки в корпусе, получают высокий вес.

**2. Чем SBERT отличается от TF-IDF?**

**Ответ:**
- **TF-IDF:** статистический метод, считает частоты слов, не учитывает контекст
- **SBERT:** нейросетевой метод, использует трансформер BERT для кодирования текста с учётом семантики

Главное отличие: SBERT понимает смысл ("кот" и "кошка" получат похожие векторы), TF-IDF — нет (это разные слова).

**3. Как работает алгоритм K-Means?**

**Ответ:**
1. Случайно выбираем k центроидов
2. Каждую точку относим к ближайшему центроиду
3. Пересчитываем центроиды как среднее точек кластера
4. Повторяем шаги 2-3 до сходимости

Критерий остановки: центроиды перестали двигаться или достигнут max_iter.

**4. Что такое Silhouette Score?**

**Ответ:**
Метрика качества кластеризации в диапазоне [-1, 1]:
```
s = (b - a) / max(a, b)

a = среднее расстояние до точек своего кластера
b = среднее расстояние до точек ближайшего другого кластера
```

- **s ≈ 1:** Точка далеко от других кластеров (хорошо)
- **s ≈ 0:** Точка на границе
- **s < 0:** Точка ближе к другому кластеру (плохо)

**5. Зачем нужна предобработка текста?**

**Ответ:**
- **Удаление шума:** HTML, эмодзи, пунктуация не несут смысла
- **Нормализация:** "Coffee", "coffee", "COFFEE" должны быть одним словом
- **Уменьшение размерности:** стоп-слова не различают документы
- **Объединение форм:** "running" и "run" — это одно понятие (лемматизация)

Без предобработки модель будет путаться в разных формах одного слова.

### Продвинутые вопросы

**6. Почему K-Means чувствителен к инициализации?**

**Ответ:**
K-Means — локальный алгоритм оптимизации. Начальные центроиды влияют на итоговый результат:
- Плохая инициализация → локальный минимум → плохие кластеры
- Хорошая инициализация → глобальный минимум → хорошие кластеры

Решение: `n_init=10` — запускаем алгоритм 10 раз с разными начальными точками, выбираем лучший результат (наименьший inertia).

**7. Что такое t-SNE и зачем он нужен?**

**Ответ:**
t-SNE (t-distributed Stochastic Neighbor Embedding) — алгоритм снижения размерности для визуализации.

Задача: отобразить многомерные данные (5000 или 384 измерения) в 2D так, чтобы:
- Похожие точки остались рядом
- Далёкие точки остались далеко

Использует вероятностные распределения и оптимизацию KL-дивергенции. Хорош для визуализации, но не для ML (не сохраняет глобальную структуру).

**8. Чем Davies-Bouldin Index отличается от Silhouette Score?**

**Ответ:**
**Silhouette Score:**
- Смотрит на каждую точку индивидуально
- Сравнивает расстояние до своего и чужого кластера
- Диапазон: [-1, 1], больше — лучше

**Davies-Bouldin Index:**
- Смотрит на кластеры целиком
- Сравнивает компактность кластера с расстоянием между центроидами
- Диапазон: [0, ∞), меньше — лучше

DB быстрее вычисляется (только центроиды), Silhouette точнее (все попарные расстояния).

**9. Как KeyBERT извлекает ключевые слова?**

**Ответ:**
1. Текст кодируется в вектор с помощью SBERT
2. Все слова и фразы (n-граммы) тоже кодируются в векторы
3. Вычисляется cosine similarity между текстом и каждой фразой
4. Фразы с наибольшей похожестью = ключевые слова

Параметр `diversity` использует Max Sum Distance для разнообразия: не все топ-слова — синонимы первого слова.

**10. Как бы вы улучшили кластеризацию?**

**Ответ:**
1. **Больше данных:** использовать весь датасет (568K вместо 15K)
2. **Другие алгоритмы:** DBSCAN (не нужно указывать k), Hierarchical Clustering
3. **Гибридный подход:** объединить TF-IDF и SBERT (конкатенация векторов)
4. **Тонкая настройка SBERT:** дообучить модель на наших данных (fine-tuning)
5. **Тематическое моделирование:** LDA для выделения тем
6. **Учёт рейтингов:** использовать Score (1-5 звёзд) как дополнительный признак
7. **Dimensionality reduction:** PCA/UMAP перед кластеризацией (ускорение + шумоподавление)
