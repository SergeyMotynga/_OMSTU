# Конспект для защиты лабораторной работы №3

Файл относится к ноутбуку `lab_3_artificial_ant.ipynb`.

Тема работы: эволюционная программа для поиска конечного автомата, который управляет искусственным муравьем на поле `32 x 32` и собирает как можно больше еды.

## Коротко о решении

В работе каждая особь популяции является конечным автоматом. Автомат хранится как геном: список чисел. Каждое число кодирует два параметра:

- действие муравья;
- следующее состояние автомата.

Муравей видит не все поле, а только ситуацию впереди:

- впереди еда;
- впереди препятствие или граница;
- впереди пустая клетка.

После этого автомат выбирает действие:

- `Вперед`;
- `Назад`;
- `Налево`;
- `Направо`.

Фитнес-функция равна количеству собранной еды. Чем больше еды собрал муравей, тем лучше автомат.

## Соответствие заданию

Работа соответствует основным требованиям:

- поле имеет размер `32 x 32`;
- на поле есть еда и препятствия;
- начальная позиция муравья задается как `(0, 0)`;
- особи представлены конечными автоматами;
- используется эволюционный алгоритм с популяцией автоматов;
- фитнес равен количеству собранной еды;
- есть элитизм;
- есть мутация правил переходов;
- есть турнирный отбор и скрещивание;
- выводится лучший результат;
- выводится таблица переходов лучшего автомата;
- строится путь муравья;
- строятся графики сходимости и разнообразия;
- результаты сохраняются в папку `results`.

В задании сказано, что генетический алгоритм не является эволюционным. В ноутбуке по сути используется эволюционный подход с популяцией, отбором, элитизмом, скрещиванием и мутацией. На защите лучше говорить: "используется эволюционный алгоритм с генетическими операторами".

## Классы

Классов в программе нет.

Вся логика реализована функциями и обычными структурами данных:

- поле хранится как `numpy.ndarray`;
- геном хранится как `list[int]`;
- конечный автомат после декодирования хранится как список строк переходов;
- результаты прогонов хранятся в словарях `dict`.

## Функции

### `generate_grid()`

Создает поле `32 x 32`.

Что делает:

- создает пустую матрицу;
- случайно размещает препятствия;
- случайно размещает еду кластерами;
- очищает стартовую клетку `(0, 0)`;
- возвращает готовое поле.

### `create_random_genome()`

Создает случайный геном конечного автомата.

Геном состоит из `NUM_STATES * NUM_INPUTS` чисел. В этой работе это `10 * 3 = 30` чисел.

Каждое число кодирует:

- одно из четырех действий;
- следующее состояние автомата.

### `decode_genome(genome)`

Преобразует список чисел в таблицу переходов конечного автомата.

Для каждой пары "состояние + входной сигнал" достается число из генома. Потом оно разделяется на:

- `action = value // NUM_STATES`;
- `next_state = value % NUM_STATES`.

### `simulate_ant(genome, grid)`

Запускает муравья на поле.

Что делает:

- декодирует геном в конечный автомат;
- ставит муравья в клетку `(0, 0)`;
- задает начальное направление вверх;
- на каждом шаге смотрит на клетку впереди;
- определяет входной сигнал для автомата;
- получает действие и следующее состояние;
- выполняет движение или поворот;
- собирает еду при попадании на клетку с едой;
- возвращает количество еды и путь.

### `calculate_diversity(population)`

Считает генетическое разнообразие популяции.

Используется нормированное расстояние Хэмминга: сравниваются гены разных особей, считается доля отличающихся генов, затем усредняется по всем парам.

### `run_evolution(fixed_grid)`

Запускает один полный прогон эволюционного алгоритма.

Что делает:

- создает начальную популяцию случайных автоматов;
- оценивает каждую особь через `simulate_ant`;
- сортирует особей по фитнесу;
- сохраняет лучшую найденную особь;
- записывает историю лучшего фитнеса, среднего фитнеса и разнообразия;
- переносит элиту в новое поколение;
- выбирает родителей турнирным отбором;
- создает потомка одноточечным скрещиванием;
- мутирует отдельные гены;
- повторяет процесс `GENERATIONS` поколений.

### `genome_to_table(genome)`

Преобразует геном лучшего автомата в удобную таблицу для вывода.

В таблице строки соответствуют состояниям `S0 ... S9`, а столбцы соответствуют входным сигналам.

### `visualize_path(grid, path, food_collected, total_food)`

Рисует поле, препятствия, еду и путь муравья.

На графике:

- белый цвет — пустые клетки;
- красный цвет — препятствия;
- зеленый цвет — еда;
- синяя линия — путь муравья;
- синяя точка — старт;
- фиолетовая точка — финиш.

### `plot_convergence_aggregated(all_results, total_food)`

Строит график сходимости по всем прогонам.

Показывает:

- среднее значение лучшей особи по поколениям;
- среднее значение популяции по поколениям;
- разброс `±1 sigma`;
- горизонтальную линию общего количества еды.

### `plot_diversity_aggregated(all_results)`

Строит график генетического разнообразия.

Если разнообразие падает, значит особи становятся похожими: эволюция закрепляет удачные решения.
