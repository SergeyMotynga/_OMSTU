# Конспект для защиты лабораторной работы №3

Тема: эволюционная программа для решения задачи искусственного муравья.

Этот файл написан как шпаргалка: сначала общий смысл, потом функции, потом код в формате "строка кода → что она делает".

## Общая идея

В лабораторной работе муравьем управляет конечный автомат. Одна особь в популяции — это один автомат.

Автомат получает простой входной сигнал:

- впереди еда;
- впереди препятствие или граница;
- впереди пустая клетка.

После этого автомат выбирает действие:

- идти вперед;
- идти назад;
- повернуть налево;
- повернуть направо.

Фитнес равен количеству собранной еды. Чем больше еды собрал муравей, тем лучше особь.

## Классы

Классов в программе нет.

Вместо классов используются:

- функции;
- списки;
- словари;
- массивы `numpy`.

Если спросят, почему нет классов, можно сказать: задача небольшая, поэтому конечный автомат удобно хранить как список переходов, а поле как матрицу `numpy`.

`import random`

Подключаем стандартный генератор случайных чисел. Он используется для создания поля, геномов, турниров, скрещивания и мутаций.

`from pathlib import Path`

Подключаем удобную работу с путями к файлам. Через `Path` создается папка `results` и сохраняются результаты.

`import matplotlib.pyplot as plt`

Подключаем библиотеку для построения графиков.

Подключаем `numpy`. Он нужен для матрицы поля, подсчета еды и статистики.

`GENERATIONS = 90`

Количество поколений в одном прогоне эволюции.

`MUTATION_RATE = 0.045`

`ELITE_COUNT = 2`

Количество лучших особей, которые переходят в следующее поколение без изменений.

`TOURNAMENT_SIZE = 5`

Размер турнира при отборе. Случайно берется 5 особей, и лучшая из них становится родителем.

`RUN_COUNT = 20`

Количество независимых запусков эволюции. Это нужно, чтобы оценить устойчивость результата.

`RANDOM_SEED = 137`

Фиксирует случайность. Благодаря этому при повторном запуске можно получить такие же результаты.

`OBSTACLE_DENSITY = 0.12`

Плотность препятствий. Примерно 12% клеток пытаются стать препятствиями.

`FOOD_CLUSTER_COUNT = 21`

Количество кластеров еды на поле.

`FOOD_CLUSTER_RADIUS = 3`

Радиус одного кластера еды.

`FOOD_CLUSTER_PROBABILITY = 0.46`

Вероятность, что клетка внутри кластера станет едой.

`EMPTY = 0`

Код пустой клетки.

`OBSTACLE = 1`

Код препятствия.

`FOOD = 2`

Код еды.

`ACTIONS = ["Вперед", "Назад", "Налево", "Направо"]`

Список названий действий. Нужен для красивого вывода таблицы переходов.

`INPUTS = ["Еда впереди", "Препятствие впереди", "Пусто впереди"]`

Список названий входных сигналов. Нужен для заголовков таблицы переходов.

Фиксируем случайность для модуля `random`.

`np.random.seed(RANDOM_SEED)`

Фиксируем случайность для `numpy`.

## Код по строкам: генерация поля

`def generate_grid() -> np.ndarray:`

Создаем функцию генерации поля. Она возвращает массив `numpy`.

`"""Создает поле с препятствиями и кластерами еды."""`

Краткое описание функции.

`grid = np.zeros((GRID_SIZE, GRID_SIZE), dtype=int)`

Создаем матрицу `32 x 32`, заполненную нулями. Нули означают пустые клетки.

`obstacle_count = int(GRID_SIZE**2 * OBSTACLE_DENSITY)`

Считаем, сколько препятствий надо разместить. Берем площадь поля и умножаем на плотность препятствий.

`for _ in range(obstacle_count):`

Запускаем цикл размещения препятствий.

`row = random.randint(0, GRID_SIZE - 1)`

Случайно выбираем строку.

`col = random.randint(0, GRID_SIZE - 1)`

Случайно выбираем столбец.

`if (row, col) != (0, 0):`

Проверяем, что выбранная клетка не является стартом муравья.

`grid[row, col] = OBSTACLE`

Ставим препятствие в выбранную клетку.

`for _ in range(FOOD_CLUSTER_COUNT):`

Запускаем цикл создания кластеров еды.

`center_row = random.randint(FOOD_CLUSTER_RADIUS, GRID_SIZE - FOOD_CLUSTER_RADIUS - 1)`

Случайно выбираем строку центра кластера так, чтобы кластер не выходил за границы поля.

`center_col = random.randint(FOOD_CLUSTER_RADIUS, GRID_SIZE - FOOD_CLUSTER_RADIUS - 1)`

Случайно выбираем столбец центра кластера.

`for d_row in range(-FOOD_CLUSTER_RADIUS, FOOD_CLUSTER_RADIUS + 1):`

Перебираем смещения по строкам вокруг центра кластера.

`for d_col in range(-FOOD_CLUSTER_RADIUS, FOOD_CLUSTER_RADIUS + 1):`

Перебираем смещения по столбцам вокруг центра кластера.

`row = center_row + d_row`

Считаем реальную строку текущей клетки внутри кластера.

`col = center_col + d_col`

Считаем реальный столбец текущей клетки внутри кластера.

`if 0 <= row < GRID_SIZE and 0 <= col < GRID_SIZE and grid[row, col] == EMPTY and random.random() < FOOD_CLUSTER_PROBABILITY:`

Проверяем четыре условия: клетка внутри поля, клетка пустая, и случайная вероятность разрешила положить туда еду.

`grid[row, col] = FOOD`

Размещаем еду в клетке.

`grid[0, 0] = EMPTY`

Очищаем стартовую клетку, чтобы муравей не начинал в препятствии или еде.

`return grid`

Возвращаем готовое поле.

`original_grid = generate_grid()`

Создаем поле один раз. Все 20 прогонов используют одинаковое поле, чтобы результаты можно было честно сравнивать.

`total_food = int(np.count_nonzero(original_grid == FOOD))`

Считаем общее количество еды на поле.

`print(f"Всего еды: {total_food}")`

Выводим количество еды.

## Код по строкам: геном и конечный автомат

`def create_random_genome() -> list[int]:`

Создаем функцию генерации случайного генома.

`max_gene_value = (NUM_ACTIONS - 1) * NUM_STATES + (NUM_STATES - 1)`

Считаем максимальное значение гена. При 4 действиях и 10 состояниях возможны значения от 0 до 39.

`return [random.randint(0, max_gene_value) for _ in range(NUM_STATES * NUM_INPUTS)]`

Создаем список из 30 случайных чисел. Почему 30: 10 состояний умножаем на 3 входных сигнала.

`def decode_genome(genome: list[int]) -> list[list[tuple[int, int]]]:`

Создаем функцию, которая превращает геном в таблицу переходов автомата.

`fsm = []`

Создаем пустой список для конечного автомата.

`for state in range(NUM_STATES):`

Перебираем все состояния автомата.

`row = []`

Создаем строку таблицы переходов для текущего состояния.

`for input_type in range(NUM_INPUTS):`

Перебираем все входные сигналы.

`value = genome[state * NUM_INPUTS + input_type]`

Достаем нужный ген. Индекс считается по формуле: номер состояния умножить на число входов и прибавить номер входа.

`action = value // NUM_STATES`

Получаем действие. Целочисленное деление показывает, к какой группе действий относится число.

`next_state = value % NUM_STATES`

Получаем следующее состояние. Остаток от деления всегда будет от 0 до 9.

`row.append((action, next_state))`

Добавляем в строку таблицы пару: действие и следующее состояние.

`fsm.append(row)`

Добавляем строку переходов в автомат.

`return fsm`

Возвращаем конечный автомат.

## Код по строкам: симуляция муравья

`def simulate_ant(genome: list[int], grid: np.ndarray) -> tuple[int, list[tuple[int, int]]]:`

Создаем функцию запуска муравья. Она возвращает количество еды и путь.

`fsm = decode_genome(genome)`

Сначала преобразуем геном в таблицу переходов.

`row, col = 0, 0`

Муравей начинает в клетке `(0, 0)`.

`direction = 0`

Начальное направление — вверх.

`state = 0`

Начальное состояние автомата — `S0`.

`path = [(row, col)]`

Создаем путь и сразу добавляем стартовую клетку.

`food_collected = 0`

Счетчик собранной еды сначала равен нулю.

`moves = [(-1, 0), (0, 1), (1, 0), (0, -1)]`

Список направлений: вверх, вправо, вниз, влево.

`if grid[row, col] == FOOD:`

Проверяем, есть ли еда на стартовой клетке.

`grid[row, col] = EMPTY`

Если еда есть, убираем ее.

`food_collected += 1`

Увеличиваем счетчик еды.

`for _ in range(MAX_STEPS):`

Запускаем цикл шагов муравья.

`front_row = row + moves[direction][0]`

Считаем строку клетки перед муравьем.

`front_col = col + moves[direction][1]`

Считаем столбец клетки перед муравьем.

`if front_row < 0 or front_row >= GRID_SIZE or front_col < 0 or front_col >= GRID_SIZE or grid[front_row, front_col] == OBSTACLE:`

Проверяем, что впереди: граница поля или препятствие.

`sensor = 1`

Если впереди граница или препятствие, входной сигнал автомата равен `1`.

`elif grid[front_row, front_col] == FOOD:`

Если впереди не препятствие, проверяем, есть ли там еда.

`sensor = 0`

Если впереди еда, входной сигнал равен `0`.

`else:`

Иначе впереди обычная пустая клетка.

`sensor = 2`

Для пустой клетки входной сигнал равен `2`.

`action, next_state = fsm[state][sensor]`

Берем из автомата действие и следующее состояние для текущего состояния и текущего сигнала.

`state = next_state`

Переводим автомат в следующее состояние.

`if action in (0, 1):`

Проверяем, является ли действие перемещением. `0` — вперед, `1` — назад.

`move_direction = direction if action == 0 else (direction + 2) % 4`

Если действие вперед, используем текущее направление. Если назад, разворачиваем направление на 180 градусов.

`target_row = row + moves[move_direction][0]`

Считаем строку целевой клетки.

`target_col = col + moves[move_direction][1]`

Считаем столбец целевой клетки.

`if 0 <= target_row < GRID_SIZE and 0 <= target_col < GRID_SIZE and grid[target_row, target_col] != OBSTACLE:`

Проверяем, что целевая клетка внутри поля и не является препятствием.

`row, col = target_row, target_col`

Перемещаем муравья в целевую клетку.

`path.append((row, col))`

Добавляем новую клетку в путь.

`if grid[row, col] == FOOD:`

Проверяем, была ли в новой клетке еда.

`grid[row, col] = EMPTY`

Если еда была, удаляем ее.

`food_collected += 1`

Увеличиваем счетчик собранной еды.

`elif action == 2:`

Если действие равно `2`, это поворот налево.

`direction = (direction - 1) % 4`

Меняем направление на одно против часовой стрелки.

`elif action == 3:`

Если действие равно `3`, это поворот направо.

`direction = (direction + 1) % 4`

Меняем направление на одно по часовой стрелке.

`return food_collected, path`

Возвращаем количество собранной еды и путь муравья.

## Код по строкам: разнообразие популяции

`def calculate_diversity(population: list[list[int]]) -> float:`

Создаем функцию подсчета разнообразия популяции.

`if len(population) < 2:`

Проверяем, есть ли хотя бы две особи.

`return 0.0`

Если особей меньше двух, сравнивать нечего, поэтому разнообразие равно нулю.

`total_difference = 0.0`

Создаем переменную для суммы различий.

`comparisons = 0`

Создаем счетчик сравнений.

`genome_length = len(population[0])`

Запоминаем длину генома.

`for i in range(len(population)):`

Берем первую особь для сравнения.

`for j in range(i + 1, len(population)):`

Берем вторую особь. Начинаем с `i + 1`, чтобы не сравнивать пару дважды.

`difference = sum(a != b for a, b in zip(population[i], population[j]))`

Считаем, сколько генов отличается у двух особей.

`total_difference += difference / genome_length`

Нормируем различие на длину генома и добавляем к общей сумме.

`comparisons += 1`

Увеличиваем счетчик сравнений.

`return total_difference / comparisons`

Возвращаем среднее различие по всем парам особей.

## Код по строкам: эволюционный алгоритм

`def run_evolution(fixed_grid: np.ndarray) -> dict[str, object]:`

Создаем функцию одного прогона эволюционного алгоритма.

`population = [create_random_genome() for _ in range(POPULATION_SIZE)]`

Создаем начальную популяцию из 90 случайных геномов.

`best_ever_genome = None`

Создаем переменную для лучшего генома за весь прогон.

`best_ever_path = []`

Создаем переменную для пути лучшего муравья.

`best_ever_fitness = -1`

Лучший фитнес сначала равен `-1`, чтобы первая оцененная особь точно стала лучше.

`fitness_history = []`

Список для истории лучшего фитнеса по поколениям.

`avg_fitness_history = []`

Список для истории среднего фитнеса по поколениям.

`diversity_history = []`

Список для истории разнообразия по поколениям.

`for generation in range(GENERATIONS):`

Запускаем цикл поколений.

`results = []`

Создаем список результатов текущего поколения.

`for genome in population:`

Перебираем каждый геном в популяции.

`grid_copy = fixed_grid.copy()`

Копируем поле для каждой особи. Это важно, потому что еда во время симуляции исчезает.

`fitness, path = simulate_ant(genome, grid_copy)`

Запускаем муравья и получаем количество еды и путь.

`results.append((fitness, path, genome))`

Сохраняем результат особи.

`results.sort(key=lambda item: item[0], reverse=True)`

Сортируем особей по фитнесу от лучшей к худшей.

`best_fitness, best_path, best_genome = results[0]`

Берем лучшую особь текущего поколения.

`avg_fitness = float(np.mean([item[0] for item in results]))`

Считаем средний фитнес популяции.

`if best_fitness > best_ever_fitness:`

Проверяем, стала ли лучшая особь текущего поколения лучшей за весь прогон.

`best_ever_fitness = best_fitness`

Обновляем лучший фитнес.

`best_ever_path = best_path.copy()`

Копируем путь лучшего муравья.

`best_ever_genome = best_genome.copy()`

Копируем лучший геном.

`fitness_history.append(best_fitness)`

Добавляем лучший фитнес поколения в историю.

`avg_fitness_history.append(avg_fitness)`

Добавляем средний фитнес поколения в историю.

`diversity_history.append(calculate_diversity([item[2] for item in results]))`

Считаем и сохраняем разнообразие популяции.

`new_population = [copy.deepcopy(item[2]) for item in results[:ELITE_COUNT]]`

Создаем новое поколение и сразу переносим туда элиту. Это две лучшие особи без изменений.

`while len(new_population) < POPULATION_SIZE:`

Пока новое поколение не заполнено, создаем потомков.

`contenders = random.sample(results, TOURNAMENT_SIZE)`

Случайно выбираем участников турнира для первого родителя.

`parent_a = max(contenders, key=lambda item: item[0])[2]`

Берем лучшего участника турнира как первого родителя.

`contenders = random.sample(results, TOURNAMENT_SIZE)`

Случайно выбираем участников турнира для второго родителя.

`parent_b = max(contenders, key=lambda item: item[0])[2]`

Берем лучшего участника турнира как второго родителя.

`child = parent_a.copy()`

Создаем потомка как копию первого родителя.

`crossover_point = random.randint(0, len(child) - 1)`

Случайно выбираем точку скрещивания.

`child[crossover_point:] = parent_b[crossover_point:]`

Заменяем хвост потомка хвостом второго родителя.

`max_gene_value = (NUM_ACTIONS - 1) * NUM_STATES + (NUM_STATES - 1)`

Считаем максимальное допустимое значение гена.

`for gene_index in range(len(child)):`

Перебираем все гены потомка.

`if random.random() < MUTATION_RATE:`

Случайно решаем, мутирует ли текущий ген.

`child[gene_index] = random.randint(0, max_gene_value)`

Если ген мутирует, заменяем его новым случайным допустимым значением.

`new_population.append(child)`

## Что важно сказать про движение назад

Движение назад добавлено в программу полностью.

`NUM_ACTIONS = 4`

Теперь действий четыре, а не три.

`ACTIONS = ["Вперед", "Назад", "Налево", "Направо"]`

В список действий добавлено `Назад`.

`if action in (0, 1):`

Оба действия `Вперед` и `Назад` считаются перемещением.

`move_direction = direction if action == 0 else (direction + 2) % 4`

Если действие `Вперед`, муравей идет по текущему направлению. Если действие `Назад`, он идет в противоположную сторону.

В таблице лучшего автомата действие `Назад` реально встречается, например:

`S5 | Препятствие впереди | Назад -> S2`

Это значит: если автомат находится в состоянии `S5` и впереди препятствие, муравей идет назад и автомат переходит в состояние `S2`.

## Частые вопросы

`Зачем grid.copy()?`

Потому что при симуляции еда исчезает. Если не копировать поле для каждой особи, первая особь изменит поле, и следующие будут оцениваться нечестно.

`Что такое фитнес?`

Фитнес — это качество особи. Здесь фитнес равен количеству собранной еды.

`Что такое элитизм?`

Элитизм — это сохранение нескольких лучших особей без изменений в следующее поколение.

`Что такое турнирный отбор?`

Случайно выбирается несколько особей, и лучшая из них становится родителем.

`Что такое мутация?`

Мутация случайно меняет отдельные гены, то есть отдельные правила переходов автомата.

`Что такое ±1 sigma?`

Это одно стандартное отклонение. Оно показывает, насколько сильно отличаются результаты разных прогонов.

`Почему путь меньше MAX_STEPS?`

Потому что в путь записываются только перемещения по клеткам. Повороты налево и направо не добавляют новую клетку.
